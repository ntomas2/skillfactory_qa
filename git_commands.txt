- git status - команда получения статуса файлов в репозитории;
- git add - команда, чтобы файл попал под версионный контроль и впоследствии в нём можно было зафиксировать измененияж;

- git add . - добавить все изменённые и неотслеживаемые файлы в подготовленные;
- git commit -m - команда фиксации изменений с добавлением сообщения в кавычках после параметра"-m";
- git push - команда, которую нужно использовать для публикации изменений на сервере;
- git log - По умолчанию (без аргументов) git log перечисляет коммиты, сделанные в репозитории в обратном к хронологическому порядке: последние коммиты находятся вверху;
- git branch <branch_name> - Ветка в Git — это подвижный указатель на один из коммитов. Обычно ветка указывает на последний коммит в цепочке коммитов. Ветка берёт своё начало от какого-то одного коммита;
- git branch --list - выведет список веток, которые существуют в вашем локальном репозитории. Текущая ветка master будет выделена зелёным цветом;
- git branch --list -v - выведет информацию о последних коммитах каждой из веток;
- git checkout <branch_name> - команда для переключения на созданную ветку;
